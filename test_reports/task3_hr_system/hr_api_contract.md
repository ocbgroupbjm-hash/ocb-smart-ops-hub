# HR Enterprise System API Contract

## Version: 1.0.0
## Base URL: /api/hr

---

## Authentication
All endpoints require JWT Bearer token in Authorization header.

---

## 1. Employee Master

### GET /employees
List employees with filtering.

**Query Parameters:**
- `status`: active, inactive, resigned (optional)
- `department_id`: UUID (optional)
- `branch_id`: UUID (optional)
- `search`: string (optional)
- `page`: integer (default: 1)
- `limit`: integer (default: 50)

**Response:**
```json
{
  "employees": [...],
  "total": 100,
  "page": 1,
  "limit": 50,
  "total_pages": 2
}
```

### POST /employees
Create new employee.

**Request Body:**
```json
{
  "employee_id": "EMP-001",
  "full_name": "Ahmad Suhendra",
  "email": "ahmad@company.com",
  "department_id": "uuid",
  "position_id": "uuid",
  "join_date": "2024-01-15",
  "salary_base": 15000000
}
```

### GET /employees/{id}
Get single employee by ID or NIK.

### PUT /employees/{id}
Update employee data.

### DELETE /employees/{id}
Soft delete (set status = inactive).

## 2. Attendance

### POST /attendance/checkin
Record check-in.

**Request Body:**
```json
{
  "employee_id": "uuid or NIK",
  "method": "manual",
  "location": "Kantor Pusat",
  "latitude": -6.123,
  "longitude": 106.123,
  "shift_id": "uuid"
}
```

**Response:**
```json
{
  "status": "success",
  "check_in_time": "08:30:00",
  "is_late": true,
  "late_minutes": 30
}
```

### POST /attendance/checkout
Record check-out.

**Request Body:**
```json
{
  "employee_id": "uuid",
  "method": "manual",
  "overtime_hours": 1.5
}
```

### GET /attendance/today
Today's attendance summary.

**Response:**
```json
{
  "date": "2026-03-14",
  "summary": {
    "total_employees": 100,
    "present": 95,
    "absent": 5,
    "checked_out": 80,
    "late": 10
  }
}
```

### GET /attendance/report
Period report with date range.

**Query Parameters:**
- `start_date`: YYYY-MM-DD (required)
- `end_date`: YYYY-MM-DD (required)
- `employee_id`: UUID (optional)
- `department_id`: UUID (optional)

## 4. Payroll Engine

### GET /payroll/components
List payroll components.

### POST /payroll/run
Run payroll calculation.

**Request Body:**
```json
{
  "period_month": 3,
  "period_year": 2026,
  "branch_id": "uuid",
  "department_id": "uuid",
  "employee_ids": ["uuid1", "uuid2"]
}
```

**Response:**
```json
{
  "status": "success",
  "batch_no": "PAY-20260314-0007",
  "batch_id": "uuid",
  "summary": {
    "employee_count": 21,
    "total_gross": 115500000,
    "total_net": 113352750
  }
}
```

### POST /payroll/post/{batch_id}
Post payroll and create journal.

**Response:**
```json
{
  "status": "success",
  "batch_no": "PAY-20260314-0007",
  "journal_no": "JV-20260314-0014"
}
```

### GET /payroll/{period}
Get payroll data for period (format: YYYY-MM).

### GET /payroll/slip/{payroll_id}
Get detailed payslip with all items.

## Error Responses

All endpoints return standard error format:
```json
{
  "detail": "Error message description"
}
```

HTTP Status Codes:
- 200: Success
- 400: Bad Request (validation error)
- 401: Unauthorized
- 404: Not Found
- 500: Internal Server Error
