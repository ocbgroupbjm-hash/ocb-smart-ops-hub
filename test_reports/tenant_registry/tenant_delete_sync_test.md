## SYNC MECHANISM

### Source of Truth
```
_tenant_metadata collection in each tenant database
↓
tenant_registry reads from all databases
↓
/api/business/list returns only ACTIVE tenants
↓
Login page displays only active tenants
```

### Automatic Sync Points
1. **Tenant Create:** New database → New _tenant_metadata → Auto-visible
2. **Tenant Edit:** Update _tenant_metadata → Auto-updated
3. **Tenant Delete:** Drop database / Update status → Auto-hidden
4. **Tenant Deactivate:** Update status = inactive → Auto-hidden
